## Prerequisites

- Docker (required for local runtime only)
- One model path:
  - Esprit Cloud login (`esprit provider login esprit`), or
  - Local provider credentials (OAuth/API key), or
  - Public OpenCode free model (no auth when available)

## Installation

<Tabs>
  <Tab title="curl">
    ```bash
    curl -fsSL https://raw.githubusercontent.com/esprit-cli/Esprit/main/scripts/install.sh | bash
    ```
  </Tab>
  <Tab title="pipx">
    ```bash
    pipx install esprit-agent
    ```
  </Tab>
  <Tab title="npm">
    ```bash
    npm install -g esprit-cli@latest
    ```
  </Tab>
</Tabs>

## Configuration

Connect a provider (or use Esprit Cloud):

```bash
esprit provider login esprit
# or
esprit provider login openai
# or
esprit provider login opencode
```

<Tip>
You can run `esprit` with no arguments to open the interactive launchpad and configure provider, model, target, and scan mode in one flow.
</Tip>

## Run Your First Scan

```bash
# Interactive onboarding + launchpad
esprit

# Direct scan command
esprit scan ./your-app
```

<Note>
In local mode, the first run pulls the Docker sandbox image automatically. Esprit Cloud mode does not require Docker. Results are saved to `esprit_runs/<run-name>`.
</Note>

## Target Types

Esprit accepts multiple target types:

```bash
# Local codebase
esprit scan ./app-directory

# GitHub repository
esprit scan https://github.com/org/repo

# Live web application
esprit scan https://your-app.com

# Multiple targets (white-box testing)
esprit scan https://github.com/org/repo https://your-app.com
```

## OpenCode Public Models

When OpenCode public models are available on your machine, you can use them without API-key login.

Recommended public models:

- `opencode/minimax-m2.5-free`
- `opencode/kimi-k2.5-free`
- `opencode/gpt-5-nano`

If one public model is rate-limited during runtime, Esprit automatically falls back to another public model.

## Basic Usage

```bash
esprit scan <target> [options]
```

```bash
# Open interactive launchpad
esprit
```

## First-Run Onboarding

On first use, Esprit opens a one-time onboarding wizard before normal command flow.

- Distinct onboarding UI with theme/provider/model setup
- Runs before Docker checks
- Can be skipped and will be shown again until completed

## Interactive Launchpad

Running `esprit` with no arguments opens the launchpad.

- Select provider, model, target, and scan mode interactively
- Local-path target input supports directory autocomplete
- Selected theme is reused in the scanning TUI

## Options

<ParamField path="--target, -t" type="string" required>
  Target to test. Accepts URLs, repositories, local directories, domains, or IP addresses. Can be specified multiple times.
</ParamField>

<ParamField path="--instruction" type="string">
  Custom instructions for the scan. Use for credentials, focus areas, or specific testing approaches.
</ParamField>

<ParamField path="--instruction-file" type="string">
  Path to a file containing detailed instructions.
</ParamField>

<ParamField path="--scan-mode, -m" type="string" default="deep">
  Scan depth: `quick`, `standard`, or `deep`.
</ParamField>

<ParamField path="--non-interactive, -n" type="boolean">
  Run in headless mode without TUI. Ideal for CI/CD.
</ParamField>

<ParamField path="--config" type="string">
  Path to a custom config file (JSON) to use instead of `~/.esprit/cli-config.json`.
</ParamField>

## Examples

```bash
# Basic scan
esprit scan https://example.com

# Authenticated testing
esprit scan https://app.com --instruction "Use credentials: user:pass"

# Focused testing
esprit scan api.example.com --instruction "Focus on IDOR and auth bypass"

# CI/CD mode
esprit scan ./ -n --scan-mode quick

# Multi-target white-box testing
esprit scan https://github.com/org/app https://staging.example.com
```

## Provider Commands

```bash
esprit provider login
esprit provider status
esprit provider logout <provider>
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Scan completed, no vulnerabilities found |
| 2 | Vulnerabilities found (headless mode only) |

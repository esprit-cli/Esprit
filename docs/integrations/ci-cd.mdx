---
title: "CI/CD Integration"
description: "Run Esprit in any CI/CD pipeline"
---

Esprit runs in headless mode for automated pipelines.

## Headless Mode

Use the `-n` or `--non-interactive` flag:

```bash
esprit scan ./app -n -m quick
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | No vulnerabilities found |
| 1 | Execution error |
| 2 | Vulnerabilities found |

## GitLab CI

```yaml .gitlab-ci.yml
security-scan:
  image: docker:latest
  services:
    - docker:dind
  variables:
    ESPRIT_LLM: $ESPRIT_LLM
    LLM_API_KEY: $LLM_API_KEY
  script:
    - curl -fsSL https://raw.githubusercontent.com/esprit-cli/Esprit/main/scripts/install.sh | bash
    - esprit scan ./ -n -m quick
```

## Jenkins

```groovy Jenkinsfile
pipeline {
    agent any
    environment {
        ESPRIT_LLM = credentials('esprit-llm')
        LLM_API_KEY = credentials('llm-api-key')
    }
    stages {
        stage('Security Scan') {
            steps {
                sh 'curl -fsSL https://raw.githubusercontent.com/esprit-cli/Esprit/main/scripts/install.sh | bash'
                sh 'esprit scan ./ -n -m quick'
            }
        }
    }
}
```

## CircleCI

```yaml .circleci/config.yml
version: 2.1
jobs:
  security-scan:
    docker:
      - image: cimg/base:current
    steps:
      - checkout
      - setup_remote_docker
      - run:
          name: Install Esprit
          command: curl -fsSL https://raw.githubusercontent.com/esprit-cli/Esprit/main/scripts/install.sh | bash
      - run:
          name: Run Scan
          command: esprit scan ./ -n -m quick
```

<Note>
All CI platforms require Docker access. Ensure your runner has Docker available.
</Note>
